# Your first editor

Let's build a small editor with a button that toggles bold and lights up when
the current selection is bold. It's just enough to see the two moving parts of
the library: **calling methods on the input** and **reading its state back**.

## Rendering the input

Start by dropping an `EnrichedTextInput` on the screen. It behaves like a
regular `TextInput` — give it a `style` and a `placeholder` and you have a
working editor.

```tsx
import { EnrichedTextInput } from 'react-native-enriched-html';
import { View, StyleSheet } from 'react-native';

export default function App() {
  return (
    <View style={styles.container}>
      <EnrichedTextInput
        style={styles.input}
        placeholder="Type something here..."
      />
    </View>
  );
}

const styles = StyleSheet.create({
  container: { flex: 1, justifyContent: 'center', padding: 16 },
  input: {
    fontSize: 18,
    padding: 12,
    maxHeight: 200,
    backgroundColor: 'gainsboro',
  },
});
```

## Toggling a style

The input is uncontrolled — you don't change its content through props. Instead
you call methods on it through a `ref` of type `EnrichedTextInputInstance`.

Add a ref and a button that calls `toggleBold`. Each call flips bold on the
current selection, or arms it for the next characters you type if nothing is
selected.

```tsx {2-4,9,12-15}
import { EnrichedTextInput } from 'react-native-enriched-html';
import type { EnrichedTextInputInstance } from 'react-native-enriched-html';
import { useRef } from 'react';
import { View, Button, StyleSheet } from 'react-native';

export default function App() {
  const ref = useRef<EnrichedTextInputInstance>(null);

  return (
    <View style={styles.container}>
      <EnrichedTextInput
        ref={ref}
        style={styles.input}
        placeholder="Type something here..."
      />
      <Button title="Bold" onPress={() => ref.current?.toggleBold()} />
    </View>
  );
}
```

Every other style works the same way — there's a `toggleItalic`,
`toggleUnderline`, `toggleH1`, `toggleOrderedList`, and so on.

## Reading the state back

A toolbar button should also show whether bold is _currently_ active, so the
user knows what they're about to change. The input reports this through the
`onChangeState` event, which fires whenever the active styles change — including
when the user just moves the cursor.

The payload holds an entry per style. Use `bold.isActive` to drive the button:

```tsx {8,15-19,22-26}
import { EnrichedTextInput } from 'react-native-enriched-html';
import type {
  EnrichedTextInputInstance,
  OnChangeStateEvent,
} from 'react-native-enriched-html';
import { useRef, useState } from 'react';
import { View, Button, StyleSheet } from 'react-native';

export default function App() {
  const ref = useRef<EnrichedTextInputInstance>(null);
  const [state, setState] = useState<OnChangeStateEvent | null>(null);

  return (
    <View style={styles.container}>
      <EnrichedTextInput
        ref={ref}
        style={styles.input}
        placeholder="Type something here..."
        onChangeState={(e) => setState(e.nativeEvent)}
      />
      <Button
        title={state?.bold.isActive ? 'Unbold' : 'Bold'}
        color={state?.bold.isActive ? 'green' : 'gray'}
        onPress={() => ref.current?.toggleBold()}
      />
    </View>
  );
}
```

That's the full loop: the button **acts** on the input with `toggleBold`, and
the input **reports** the result through `onChangeState`. A real toolbar is just
this repeated for every style.

:::tip

Each `onChangeState` entry carries more than `isActive`. `isBlocking` and
`isConflicting` tell you when a style can't be applied on top of the current
one. [Core concepts](/fundamentals/core-concepts) explains the full state model,
and [Supported HTML tags](/fundamentals/html-format-and-supported-tags)
lists which styles clash with each other.

:::
